# 前端视觉风格规范

## 1. 设计方向

本项目的前端视觉方向定义为 **White Editorial Workspace**。

关键词：

- 白色主题
- 纸感
- 克制
- 高级
- 桌面学习工作台
- 阅读友好
- 专业学习工具

产品不应该呈现为“AI 聊天工具”。AI 能力应像右侧教练、反馈层、解析层一样嵌入学习流程，而不是成为视觉主角。

## 2. 需要避免的视觉倾向

避免以下常见 AI 产品视觉语言：

- 紫蓝渐变
- 霓虹光效
- 大面积发光背景
- 机器人图标
- 魔法棒图标
- 星光装饰
- 聊天气泡作为主界面风格
- “AI 正在思考”这类强人格化表达
- 过度圆角
- 过多漂浮卡片
- 大片装饰性阴影

## 3. 色彩系统

### 3.1 基础色

```css
:root {
  --color-bg: #fafaf9;
  --color-surface: #ffffff;
  --color-paper: #fdfbf7;
  --color-border: #e7e5e4;

  --color-text: #0c0a09;
  --color-text-muted: #57534e;
  --color-text-subtle: #a8a29e;

  --color-primary: #1c1917;
  --color-accent: #b45309;
  --color-success: #15803d;
  --color-warning: #ca8a04;
  --color-danger: #b91c1c;
}
```

### 3.2 使用规则

- 主背景使用 `#FAFAF9`，避免冷白。
- 内容面板使用 `#FFFFFF`。
- 长文阅读区域可使用 `#FDFBF7`，形成纸张感。
- 主操作使用近黑色 `#1C1917`。
- 强调色只用于关键状态、当前选中项、重要反馈，不大面积铺开。
- 错误、纠正、成功、进步使用语义色，不使用高饱和荧光色。

## 4. 字体系统

### 4.1 推荐字体

```css
:root {
  --font-ui: Inter, -apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans GB", sans-serif;
  --font-reading: Georgia, "Times New Roman", "Songti SC", serif;
  --font-mono: "SF Mono", ui-monospace, monospace;
}
```

### 4.2 字体使用

- UI、按钮、表单、导航使用 `--font-ui`。
- 阅读正文、文章沉浸页可使用 `--font-reading`。
- 分数、标签、调试信息、技术 ID 可使用 `--font-mono`。
- 不使用童趣字体、过度圆润字体或强科技感字体。

## 5. 布局原则

应用应呈现为 macOS 桌面学习工作台。

推荐结构：

```text
左侧：导航、模块入口、历史记录
中间：核心训练区
右侧：反馈、笔记、解析、复盘
```

布局要求：

- 中间区域永远是学习任务主体。
- AI 反馈位于右侧或上下文区域，不抢占主任务。
- 阅读和写作页面要保证足够行宽和留白。
- 工具栏保持克制，不堆叠过多按钮。
- 页面不使用营销式 hero。

## 6. 组件风格

### 6.1 圆角

```css
:root {
  --radius-sm: 6px;
  --radius-md: 8px;
}
```

使用规则：

- 按钮、输入框、标签使用 `6px` 到 `8px`。
- 不使用大圆角胶囊风格作为默认视觉。
- 内容卡片最大不超过 `8px`，除非是模态或特殊浮层。

### 6.2 边框与阴影

- 默认用边框建立层级。
- 阴影只用于浮层、菜单、模态框。
- 常规内容面板使用 `1px solid #E7E5E4`。
- 不使用厚重阴影制造“卡片墙”。

### 6.3 按钮

主按钮：

- 黑底白字。
- 用于提交、开始训练、保存、生成核心结果。

次按钮：

- 白底细边框。
- 用于取消、切换、辅助操作。

危险按钮：

- 使用低饱和红色，不使用大面积红底。

### 6.4 标签

标签用于能力点、难度、状态、题型。

规则：

- 使用低饱和背景。
- 文案短，不超过一行。
- 不使用强烈彩虹标签体系。

## 7. AI 能力的视觉表达

AI 不作为品牌视觉中心，而作为学习工作流中的能力层。

推荐文案：

- 生成解析
- 查看反馈
- 优化表达
- 分析本段
- 生成练习
- 保存为表达
- 对比二稿
- 查看复盘

避免文案：

- AI 正在思考
- 魔法生成
- 智能大脑
- 和 AI 聊聊
- AI 神奇优化

推荐图标：

- 使用 Lucide 或同类线性图标。
- 可使用 `BookOpen`、`PenLine`、`Mic`、`Volume2`、`MessageSquareText`、`ClipboardCheck`、`Search`、`Bookmark`。
- 不使用机器人、魔法棒、星光作为主要图标。

## 8. 动效原则

- 动效应短、轻、明确。
- hover 和 focus 过渡控制在 `150ms` 到 `220ms`。
- 不使用光晕、粒子、漂浮装饰。
- 不使用会影响阅读和写作专注的循环动画。
- 尊重 `prefers-reduced-motion`。

## 9. 可访问性要求

- 文本对比度至少满足 WCAG AA。
- 所有可点击元素有明确 hover 和 focus 状态。
- 表单控件必须有 label，不用 placeholder 代替 label。
- 图标按钮必须有可访问名称。
- 键盘可操作路径不能断。

## 10. 页面气质参考

### 10.1 听说页

- 更像训练控制台，而不是聊天窗口。
- 对话区清晰，但不使用花哨聊天气泡。
- 录音、播放、暂停、重试使用图标按钮。
- 会话复盘以结构化列表和评分块呈现。

### 10.2 阅读页

- 以文章为中心。
- 原文排版优先，解释和翻译在侧边或行间低干扰展示。
- 阅读区可使用纸张背景色。
- 划词菜单要轻量、精准、不过度遮挡正文。

### 10.3 写作页

- 以编辑器为中心。
- 右侧显示大纲、反馈、段落建议和二稿对比。
- 批改信息按维度和段落组织。
- 不让 AI 建议覆盖用户原文。

- [ ] 没有紫蓝 AI 渐变。
- [ ] 没有机器人、魔法棒、星光作为主要视觉。
- [ ] 主要内容区域留白充足。
- [ ] 组件圆角控制在 `6px` 到 `8px`。
- [ ] 主要层级使用边框，而不是重阴影。
- [ ] 长文阅读使用适合阅读的字体和行高。
- [ ] AI 反馈文案保持学习工具语气。
- [ ] 所有图标按钮有可访问名称。
- [ ] 所有表单控件有 label。
